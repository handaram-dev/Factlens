## 1. 제품 개요

### 1.1 한줄 정의

> AI가 뉴스를 쉬운 말로 요약하고, 팩트체크 검증 태그를 붙여주며, 같은 사건에 대한 다양한 언론사 원문을 나란히 보여주는 중립적 뉴스 브리핑 서비스.

### 1.2 슬로건

**"팩트가 검증된 쉬운 뉴스 브리핑 — 진실을 보는 렌즈"**

### 1.3 핵심 문제 정의

| 문제 | 근거 데이터 | 세부 설명 |
| --- | --- | --- |
| **뉴스가 어렵다** | 20~30대 뉴스 이용률 지속 하락 | 전문용어, 배경설명 없이 던져주는 기사 → 읽다가 포기 |
| **편향이 걱정된다** | 언론수용자 조사 — 편파적 기사(22.3%) 2위 문제점 | 어떤 언론사를 보느냐에 따라 세상이 다르게 보이는 문제 |
| **가짜뉴스** | 허위/조작정보(24.6%) 1위 문제점 | 첫 기사만 기억하고 정정보도는 못 보는 구조적 문제 |

### 1.4 해결 방식

사용자가 능동적으로 검증할 필요 없이, **읽는 것만으로 팩트체크가 되는 구조**. 매일 쓰는 습관(뉴스 소비)에 팩트체크가 자연스럽게 녹아들어간다.

- **쉬운 요약:** 전문용어 없이 누구나 이해할 수 있는 뉴스 요약문 제공
- **검증 태그:** 각 뉴스에 ✅사실 / ⚠️논란 중 / ❌왜곡 태그를 자동 부여
- **다양한 시각:** 같은 사건을 보도한 모든 언론사 원문 링크를 빠짐없이 제공
- **출처 투명성:** 판별 근거와 출처 링크를 항상 노출

## 2. 시장 분석

### 2.1 시장 기회 (Gap)

- 각 기능은 따로 존재하지만, **"매일 브리핑 + 내장된 팩트체크 + 편향 해소"를 통합한 서비스는 없음**
- 네이버뉴스 이용자 하루 평균 **7.5개** 기사 소비 — 10~15개가 적정 브리핑 분량

### 2.2 핵심 교훈

**"뉴스 읽기만 하면 자동으로 팩트체크되는 경험"** 하나에 집중해야 함.

## 3. 타겟 사용자

### 3.1 1차 타겟

- **20~30대 뉴스 이탈층:** 뉴스가 어려워서 안 보거나, 유튜브/SNS로만 뉴스를 접하는 층
- **정치적 편향이 걱정되는 사람:** "내가 보는 뉴스가 편향된 건 아닌지" 불안한 사람
- 쉬운 뉴스는 좋지만, 정확성/중립성이 아쉬운 사람

### 3.2 사용자 페르소나

> **김민지 (27세, 직장인)** "뉴스는 보고 싶은데 뭘 믿어야 할지 모르겠어요. 조선일보랑 한겨레가 같은 사건을 완전 다르게 쓰잖아요. 둘 다 보기엔 시간이 없고... 누가 정리해줬으면 좋겠어요."

## 4. 핵심 기능 (MVP)

### 4.1 뉴스 카드 구조

각 뉴스 아이템은 다음 구조로 제공:

```
📰 "정부, 최저임금 1만2000원 확정"

[쉬운 요약]
내년 최저임금이 올해보다 5.1% 오른 1만2000원으로
결정됐어요. 2026년 1월부터 적용돼요.

[검증 태그] ✅ 사실 확인됨
- 고용노동부 공식 발표 (링크)
- 최저임금위원회 의결 (링크)

→ 원문 보기: 조선일보 | 한겨레 | 연합뉴스 | KBS | MBC | ...
```

### 4.2 요약 톤 가이드 (뉴닉 스타일)

- **어미:** 해요체 (~에요, ~거든요, ~했어요)
- **비유:** 어려운 개념은 일상 비유로 풀어쓰기 (예: 망 사용료 → "고속도로 통행요금 같은 거예요")
- **배경 설명:** 본론 전에 "이게 왜 중요하냐면" 맥락을 먼저 깔아주기
- **분량:** 3~5문장, 전문용어 사용 금지
- **원칙:** 의견이 아닌 사실만 전달. 감정적 표현 배제

### 4.3 기능 상세

| 기능 | 설명 | 우선순위 |
| --- | --- | --- |
| **쉬운 요약** | AI가 전문용어 없이 3~5문장으로 핵심 요약 (뉴닉 스타일 톤) | P0 (MVP 필수) |
| **검증 태그** | ✅사실 / ⚠️논란 중 / ❌왜곡 — AI가 교차검증 후 자동 부여 | P0 (MVP 필수) |
| **전체 원문 링크** | 같은 사건을 보도한 모든 언론사 원문을 빠짐없이 링크 | P0 (MVP 필수) |
| **출처 근거 링크** | 판별 근거가 된 공식 발표, 데이터 등 원출처 제공 | P0 (MVP 필수) |

### 4.4 일일 브리핑

- **분량:** 10~15개 뉴스 (네이버뉴스 평균 7.5개보다 약간 많은 수준)
- **다루는 영역:** 정치 / 경제 / 사회 / 국제 (MVP 고정, 연예·스포츠 제외)
- **생성 주기:** 하루 1회, 매일 오전 7시 고정 생성
- **선별 기준:** Google News 노출 순서(중요도 랭킹) 기준 상위 10~15개
- **접근 방식:** 로그인 없는 공개 페이지 (MVP)
- 카테고리 분류 없이 한 피드에 섞어서 제공. 카테고리 분리 및 사용자 선택은 사용자 증가 후 도입

## 5. 기술 아키텍처

### 5.1 데이터 파이프라인 (MVP)

```
[GitHub Actions — 매일 오전 7시 트리거]
        ↓
[Python 스크립트 실행]
        ↓
[Google News RSS 수집 (feedparser)]
  https://news.google.com/rss?hl=ko&gl=KR&ceid=KR:ko
  → 하루 ~30개 수집
        ↓
[연예·스포츠 필터링 → 정치/경제/사회/국제만 남김]
        ↓
[Google News 노출 순서 기준 상위 10~15개 선별]
        ↓
[각 뉴스별 AI 처리 (Gemini Flash API)]
  - 쉬운 요약 생성 (뉴닉 스타일)
  - 교차 검증 (Google Custom Search)
  - 검증 태그 부여 (✅/⚠️/❌) — 기사 단위 1개
  - 관련 언론사 원문 링크 전체 수집
        ↓
[JSON → 정적 HTML 생성]
        ↓
[Cloudflare Pages에 배포]
```

### 5.2 뉴스 수집

**Google News RSS (MVP 단일 소스):**

| 용도 | URL |
| --- | --- |
| 한국 주요뉴스 | `https://news.google.com/rss?hl=ko&gl=KR&ceid=KR:ko` |

- 하루 ~30개 아이템 제공 (정치 ~5, 경제 ~5, 사회 ~4, 국제 ~5, 연예 ~3, 스포츠 ~3 등)
- 연예·스포츠를 제외하면 ~19개 → 10~15개 선별에 충분
- Google News가 이미 중복 기사를 클러스터링 → 활용

### 5.3 기술 스택

| 영역 | 기술 | 비고 |
| --- | --- | --- |
| 파이프라인 실행 | **GitHub Actions** (스케줄 트리거) | 매일 오전 7시 실행. 무료 티어 충분 |
| 파이프라인 코드 | **Python** | feedparser로 RSS 파싱 |
| AI 요약/검증 | **Gemini Flash API** | 무료 티어 넉넉. 품질 부족 시 Claude/GPT로 교체 |
| 교차검증 웹 검색 | **Google Custom Search** | 100회/일 무료. 기사당 2~3회 제한 |
| 프론트엔드 | **정적 사이트** | 파이프라인이 JSON → HTML 생성 |
| 호스팅 | **Cloudflare Pages** | 무료, 대역폭 무제한, 한국 CDN 노드 |
| 데이터 저장 | **JSON 파일 (Git 관리)** | DB 불필요. 과거 브리핑은 JSON으로 아카이브 |

### 5.4 저작권 대응

| 전략 | 설명 |
| --- | --- |
| **링크 기반 (MVP)** | 뉴닉 방식 — "AI 요약 + 원문 링크" 제공. 원문 복제 안 함 |

> ⚠️ RSS를 통해 수집한 뉴스 콘텐츠를 그대로 노출하는 것은 저작권법 위반. 반드시 AI가 재작성한 요약 + 원문 링크 방식으로 제공해야 함.

## 6. 검증 태그 시스템

### 6.1 태그 종류

| 태그 | 의미 | 부여 조건 |
| --- | --- | --- |
| ✅ **사실 확인됨** | 핵심 주장이 공식 출처에 의해 확인됨 | 정부 발표, 공식 통계, 다수 언론 일치 |
| ⚠️ **논란 중** | 사실 여부가 아직 확정되지 않았거나 해석이 갈림 | 전문가 의견 엇갈림, 아직 공식 확인 안 됨 |
| ❌ **왜곡/오류** | 핵심 주장에 사실과 다른 부분이 포함됨 | 데이터 오류, 맥락 누락으로 인한 오해 유도 |

### 6.2 검증 프로세스

**태그 단위:** 뉴스 기사 1건당 태그 1개. 기사 내 일부라도 논란/왜곡/오류 혐의가 있으면 해당 태그로 부여.

```
1. 기사 전체의 핵심 내용을 교차검증
   - 공식 출처 (정부, 기관 발표) 확인
   - 타 언론사 보도 교차 확인
   - 원본 데이터/통계 대조
2. 기사 단위 검증 태그 1개 부여 + 판별 근거 & 출처 링크 첨부
   - 일부라도 사실과 다른 부분이 있으면 ❌ 또는 ⚠️ 부여
3. 신뢰도가 낮은 경우 ⚠️ 보수적으로 부여
```

### 6.3 투명성 원칙

- **근거 없는 태그 없음:** 모든 검증 태그에는 반드시 판별 근거 링크가 첨부
- **AI 한계 명시:** "AI 기반 자동 검증이며, 최종 판단은 독자의 몫입니다" 고지
- **오류 정정 체계:** 사용자 제보를 통한 오류 수정 프로세스 마련

## 7. 비즈니스 모델

> MVP 단계부터 수익화 가능성을 검증한다. 수익 모델 없이는 지속 불가능.

| 모델 | 설명 | 시기 |
| --- | --- | --- |
| **무료 기본** | 일일 브리핑 전체 무료 제공 | MVP |
| **후원 (Buy Me a Coffee)** | 페이지 하단에 후원 버튼. "이 브리핑이 유용했다면 커피 한 잔 사주세요" | MVP |
| **광고** | 중립성과 충돌하지 않는 광고만 게재 (예: 교육, IT, 도서 등). 정치·언론사 광고 배제 | Growth |
| **스폰서십** | "오늘의 팩트렌즈는 OO이 후원합니다" 방식. 콘텐츠 독립성 유지 | Scale |

### 7.1 광고 플랫폼 후보

| 플랫폼 | 특징 | 비고 |
| --- | --- | --- |
| **Google AdSense** | 가장 보편적. 광고 자동 매칭 | 정치/종교 카테고리 차단 설정 필수 |
| **카카오 AdFit** | 한국 트래픽 최적화. 심사 기준 낮음 | 국내 광고주 풍부 |
| **쿠팡 파트너스** | 클릭→구매 시 수익 | 사이드바/하단 배치. 뉴스 콘텐츠와 명확히 분리 |

### 7.2 광고 원칙

- **중립성 보호:** 정치 단체, 언론사, 이해관계 충돌 가능한 광고주 배제
- **정치 광고 차단:** AdSense 민감한 카테고리(정치, 종교) 차단 설정 적용
- **콘텐츠 분리:** 광고와 뉴스 콘텐츠를 명확히 구분. 네이티브 광고 금지
- **투명성:** 광고주 목록 공개

### 7.3 비용 구조 (MVP 추산)

| 항목 | 월 비용 | 비고 |
| --- | --- | --- |
| Gemini Flash API | ₩0 | 무료 티어 내 사용 (품질 부족 시 유료 모델 전환: ₩4~20만) |
| Google Custom Search | ₩0 | 100회/일 무료. 기사당 2~3회 제한 |
| GitHub Actions | ₩0 | 하루 1회 실행, 무료 티어 충분 |
| Cloudflare Pages | ₩0 | 대역폭 무제한, 월 500회 빌드 |
| **합계** | **₩0/월** | 전부 무료 티어로 시작. 유료 모델 전환 시 ₩4~20만 |

### 7.4 SNU팩트체크 실패에서 배운 수익 원칙

- 단일 펀딩원 의존 금지
- 자체 수익 모델 없이 규모 확장 금지
- MVP 단계부터 수익화 가능성 테스트 (후원 버튼으로 H5 검증)

## 10. 부록

### 10.1 참고 데이터

- 네이버뉴스 이용자 1인당 하루 평균 **7.5개** 기사 소비 (2022, 한국일보)
