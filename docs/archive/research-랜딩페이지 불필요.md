# 리서치: 팩트렌즈 공개 전략

## 1. 랜딩페이지 필요성 분석

### 1-1. 전환율 데이터

- 전용 랜딩페이지의 중앙값 전환율: **4.2%** (이커머스 기준)
- 제품 페이지(PDP)를 랜딩으로 쓸 때 전환율: **2.86%** — 약 1.5배 차이
- 상위 10% 랜딩페이지는 전환율 **20% 이상** 달성 (오퍼-타겟 핏이 맞을 때)
- 출처: [Unbounce 41,000 랜딩페이지 분석](https://unbounce.com/landing-pages/whats-a-good-conversion-rate/), [Shopify 전환율 벤치마크](https://www.shopify.com/blog/landing-page-conversion-rate/)

### 1-2. 그런데 — 트래픽 소스에 따라 결과가 다르다

- **소셜 미디어에서 유입**될 때는 PDP(=제품/콘텐츠 페이지)가 랜딩페이지보다 **전환율 2배** 높음
- **이메일에서 유입**될 때도 PDP가 근소하게 우세
- 즉, SNS/카톡으로 공유될 팩트렌즈에는 "콘텐츠 바로 보여주기"가 오히려 유리할 수 있음
- 출처: [Bidnamic 이커머스 랜딩페이지 분석](https://www.bidnamic.com/en-us/resources/ecommerce-product-discovery-pages-everything-you-need-to-know)

### 1-3. 랜딩페이지가 오히려 해가 되는 경우

| 상황 | 이유 |
|------|------|
| "보면 바로 아는 서비스" | 설명이 불필요한데 한 단계 더 거치게 하면 이탈 |
| 모바일 트래픽 비율이 높을 때 | 모바일 사용자는 클릭 1회에도 민감 (추가 탭 로딩 = 이탈) |
| 소셜 미디어가 주 유입 채널 | 소셜에서는 콘텐츠 직접 노출이 더 효과적 |
| MVP/사이드 프로젝트 | 만들 시간에 콘텐츠를 개선하는 것이 ROI 높음 |

### 1-4. 랜딩페이지가 필요한 경우

| 상황 | 이유 |
|------|------|
| 구독/가입이 핵심 전환 | 서비스 설명 -> CTA 흐름이 필수 |
| 유료 광고 트래픽 | 검색 광고 클릭 시 명확한 가치 제안이 전환에 직결 |
| 복잡한 서비스 | 왜 필요한지 설명 없이는 이해 불가 |
| B2B / 기업 타겟 | 의사결정자에게 충분한 정보 전달 필요 |

### 1-5. 팩트렌즈에 적용

팩트렌즈는 다음 특성을 가짐:
- **보면 바로 아는 서비스**: 뉴스 브리핑 + 팩트체크 태그 → 3초면 이해 가능
- **구독/가입 없음**: 이메일 수집이나 회원가입 없는 무료 정적 페이지
- **모바일 중심**: 20~30대 타겟 → 모바일 비율 높을 것
- **소셜 공유가 주 유입 채널이 될 것**: 카톡, 인스타, X 등

**결론: 별도 랜딩페이지는 현재 단계에서 불필요하며, 오히려 이탈을 유발할 위험이 있다.**

### 2-4. 벤치마크 종합 — 팩트렌즈와의 차이

| | 뉴닉/MB/theSkimm | 팩트렌즈 |
|--|---|---|
| **형태** | 이메일 뉴스레터 | 웹 정적 페이지 |
| **전환 목표** | 이메일 구독 | 없음 (방문 자체가 목표) |
| **재방문 유도** | 매일 이메일 발송 | 없음 (직접 재방문 필요) |
| **랜딩페이지** | 구독 전환용 필수 | 전환할 것이 없으므로 불필요 |
| **소셜 공유** | 랜딩으로 유도 | 콘텐츠 직접 공유가 가능 |

**핵심 인사이트**: 뉴닉/MB/theSkimm은 모두 "구독"이라는 전환이 있었기 때문에 랜딩페이지가 필수였다. 팩트렌즈는 전환 목표가 없는 무료 콘텐츠이므로, 콘텐츠 자체가 최고의 랜딩페이지 역할을 한다.

## 3. 팩트렌즈 상황에 맞는 구체적 방안 비교

### 방안 A: 별도 랜딩페이지

```
factlens.pages.dev → 랜딩페이지 (서비스 설명)
factlens.pages.dev/briefing → 실제 브리핑
```

| 장점 | 단점 |
|------|------|
| 서비스 가치를 명확히 전달 가능 | 추가 클릭 1회 → 이탈 위험 |
| 전문적이고 완성된 인상 | 구현 비용 (디자인, 개발, 유지보수) |
| 향후 구독 기능 추가 시 활용 가능 | 매일 바뀌는 콘텐츠와 정적 랜딩의 괴리 |
| SEO에 유리한 별도 페이지 | 현재 Cloudflare Pages 구조 변경 필요 |

**구현 비용**: 높음 (라우팅 구조 변경, 별도 HTML, 유지보수)
**효과**: 현재 단계에서는 낮음 (전환할 것이 없음)
**적합한 시점**: 이메일 구독, 앱 다운로드 등 전환 목표가 생길 때

### 방안 B: 브리핑 페이지 상단에 히어로 섹션 추가 (권장)

```
factlens.pages.dev → 히어로 섹션 + 브리핑 (단일 페이지)
```

| 장점 | 단점 |
|------|------|
| 추가 클릭 없이 가치 전달 + 콘텐츠 즉시 노출 | 매일 방문하는 재방문자에게는 불필요한 영역 |
| 구현 비용 최소 (HTML/CSS만 추가) | 모바일에서 스크롤 한 번 더 필요 |
| 소셜 공유 시 "이게 뭔지" 바로 인지 | 히어로 높이가 크면 콘텐츠 접근성 저하 |
| OG 태그와 시너지 | — |
| 기존 구조 변경 불필요 | — |

**구현 비용**: 낮음 (Jinja2 템플릿에 섹션 1개 추가)
**효과**: 높음 (첫 방문자에게 3초 안에 가치 전달)

**히어로 섹션 예시 구성**:
```
[팩트렌즈 로고]
"뉴스, 어렵지 않아요"
AI가 오늘의 뉴스를 쉬운 말로 요약하고
팩트체크까지 해드립니다.

↓ 오늘의 브리핑 보기
```

### 방안 C: 현재 그대로 + OG 태그/메타 설명 강화

```
factlens.pages.dev → 현재 그대로
```

| 장점 | 단점 |
|------|------|
| 구현 비용 제로 | 첫 방문자가 "이게 뭔 서비스지?" 할 수 있음 |
| 콘텐츠 즉시 노출 | 소셜 공유 시 OG 미리보기에만 의존 |
| 현재 구조 유지 | 서비스 차별점 전달 기회 부족 |

**구현 비용**: 거의 없음 (OG 태그 이미 있음)
**효과**: 콘텐츠 품질이 높다면 충분할 수 있으나, 차별화 전달이 약함

### 방안 비교 요약

| 기준 | A. 별도 랜딩 | B. 히어로 섹션 | C. 현재 유지 |
|------|:---:|:---:|:---:|
| 구현 비용 | 높음 | **낮음** | 없음 |
| 첫 방문자 이해도 | 높음 | **높음** | 보통 |
| 이탈률 위험 | 높음 | **낮음** | 낮음 |
| 소셜 공유 효과 | 보통 | **높음** | 보통 |
| 재방문자 경험 | 좋음 | 약간 불편 | **최적** |
| 확장성 | 높음 | 보통 | 낮음 |
| **종합 추천도** | 현재 부적합 | **최적** | 차선 |

## 4. "핵심 가치 전달" 방법론

### 4-1. 3초 법칙 — 첫 방문자가 즉시 이해하게 하는 방법

웹사이트 방문자는 평균 **3초 안에** 페이지를 계속 볼지 떠날지 결정한다. 히어로 섹션에서 다음 요소가 즉시 보여야 한다:

1. **헤드라인**: 짧고 이해하기 쉬운 한 줄 (가치 중심, 기능 설명 X)
2. **서브헤드**: 헤드라인을 보완하는 1~2줄
3. **CTA**: 스크롤하거나 클릭할 명확한 다음 행동

출처: [Prismic - Hero Section Best Practices](https://prismic.io/blog/website-hero-section), [LogRocket - Hero Section Examples](https://blog.logrocket.com/ux-design/hero-section-examples-best-practices/)

### 4-2. 팩트렌즈를 위한 헤드라인 후보

| 유형 | 예시 |
|------|------|
| 문제 해결형 | "뉴스, 어렵지 않아요" |
| 가치 제안형 | "쉬운 말 + 팩트체크 = 오늘의 뉴스 끝" |
| 행동 유도형 | "3분이면 오늘의 뉴스 파악 완료" |
| 타겟 호소형 | "뉴스 안 보는 당신을 위한 AI 브리핑" |
| 차별화형 | "검증된 팩트만 쉬운 말로" |

### 4-3. 뉴스 서비스에서 신뢰를 구축하는 요소

뉴스 서비스는 일반 서비스보다 **신뢰(trust)**가 더 중요하다. 팩트렌즈에 적용할 수 있는 신뢰 구축 요소:

| 요소 | 팩트렌즈 현재 상태 | 개선 방향 |
|------|---|---|
| **출처 투명성** | O (원문 보기, 출처 확인하기) | 이미 강점 — 히어로에서도 강조 |
| **검증 프로세스 설명** | X | "AI가 여러 언론사를 교차 확인합니다" 한 줄 추가 |
| **면책 고지** | O (footer에 있음) | 충분 |
| **운영자 정보** | X | 간단한 소개 또는 About 링크 |
| **소셜 프루프** | X | 초기에는 없어도 됨, 추후 방문자 수/공유 수 등 |
| **일관된 업데이트** | O (매일 자동 생성) | 히어로에 "매일 아침 업데이트" 명시 |

출처: [Trust Signals 가이드](https://www.trustsignals.com/blog/chapter-5-website-trust-signals-making-visitors-feel-at-home), [Webstacks - Trust Signals](https://www.webstacks.com/blog/trust-signals)

### 4-4. 소셜 공유 최적화

SNS에서 링크를 공유할 때 사용자가 보는 것은 **OG(Open Graph) 미리보기**다:

- **og:title**: "2026년 3월 2일 모닝 브리핑 — 팩트렌즈" (현재 OK)
- **og:description**: "팩트가 검증된 쉬운 뉴스 브리핑 — 진실을 보는 렌즈" (현재 OK, 개선 가능)
- **og:image**: **없음** — 이것이 가장 큰 약점

소셜 미디어에서 이미지가 없는 링크는 클릭률이 **현저히 낮다**. og:image를 추가하면 공유 효과가 크게 향상된다.

**og:image 방향**:
- 팩트렌즈 로고 + 날짜 + "오늘의 브리핑" 텍스트가 들어간 카드 이미지
- 정적 이미지 1장이면 충분 (매일 동적 생성은 overkill)

## 5. 종합 추천

### 현재 단계에서의 최적 전략: 방안 B (히어로 섹션 추가)

**이유**:
1. 팩트렌즈는 "보면 바로 아는 서비스"이지만, 첫 방문자에게 **왜 이 서비스가 특별한지** 3초 안에 전달할 장치가 현재 없다
2. 현재 헤더는 `팩트렌즈` + `팩트가 검증된 쉬운 뉴스 브리핑` 한 줄뿐 — 서비스의 핵심 가치(AI 요약, 팩트체크, 다양한 언론사 비교)가 드러나지 않음
3. 별도 랜딩은 전환 목표 없이 만들면 오버엔지니어링
4. 히어로 섹션은 구현 비용 최소(Jinja2 템플릿 + CSS)이면서 효과는 가장 큼
5. 소셜 공유의 주 유입 채널에서, 콘텐츠를 직접 보여주면서 상단에 맥락을 제공하는 것이 최적

### 함께 진행하면 좋은 것
1. **og:image 추가** — 소셜 공유 클릭률 향상에 가장 임팩트 큰 단일 작업
2. **히어로 섹션에 "매일 아침 업데이트" 명시** — 재방문 동기 부여
3. **히어로 섹션 접기 기능** (선택) — 재방문자가 원하면 히어로를 숨길 수 있게

### 공개 채널 전략

| 채널 | 적합도 | 이유 |
|------|:---:|------|
| **X (트위터) 한국** | 높음 | 뉴스/미디어 관심층 집중, 링크 공유 친화적 |
| **Reddit (r/korea 등)** | 보통 | 영어 설명 필요, 피드백 받기 좋음 |
| **커뮤니티 (디시, 클리앙 등)** | 높음 | 20~30대 타겟과 겹침 |
