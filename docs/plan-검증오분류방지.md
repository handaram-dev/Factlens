# Plan — 검증 오분류 방지: misleading 판정 기준 강화

**Date:** 2026-03-02
**기반:** `research-검증시스템분석.md`

---

## 문제

AI가 "검색에서 확인되지 않음"을 "사실과 다름"으로 비약하여 misleading 태그를 오부여하고 있다.

**실제 사례 — MWC26 샤오미 기사:**
- "한국이 1차 출시국에 포함" → 검색에서 확인 안 됨
- "메인홀에 대규모 부스" → 공식 출처에서 전부 확인되지 않음
- "아너 로봇폰, 샤오미 17 울트라 등 대부분의 내용은 확인됨"
- 결과: ❌왜곡/오류 → 실제로는 ⚠️사실 확인 중이 적절

**근본 원인 (리서치 결론):**
1. misleading 정의가 너무 넓다 — "내용이 다르다"가 "검색에서 안 나왔다"를 포함
2. "확인 안 됨 ≠ 사실과 다름" 구분이 프롬프트에 없다
3. "부분 미확인"일 때의 기준이 없다
4. 안전장치 규칙("확신이 없으면 unconfirmed")이 목록 4번째 줄에 묻혀 강조가 약하다
5. 문제는 오직 프롬프트에만 있고, 코드 로직은 건전하다

**위험도:** 사실인 기사에 ❌왜곡 태그 → 서비스 핵심 가치(팩트체크 신뢰)를 정면으로 훼손 → **매우 높음**

## 변경 내용

`## 판단 기준` 섹션을 전면 재구성한다. 변경 포인트 4가지:

1. **misleading 정의를 좁힌다** — "명백히 모순되는 공식 출처가 있을 때만"
2. **"확인 안 됨 ≠ 사실과 다름" 별도 경고 섹션 신설** — 가장 눈에 띄는 위치
3. **부분 미확인 기준 명시** — "일부만 미확인이면 unconfirmed"
4. **판단 우선순위 명시** — misleading은 최후의 수단

**Before (36행~42행):**
```
## 판단 기준
- verified: 공식 출처(정부, 기관, 공식 통계)로 핵심 내용이 뒷받침됨
- unconfirmed: 공식 출처를 찾을 수 없거나 충분히 확인되지 않음
- misleading: 공식 출처와 내용이 다르거나, 사실 오류 포함, 맥락 누락으로 오해 유도
- 확신이 없으면 반드시 unconfirmed으로 판단하세요
- evidence에는 검색에서 실제 비교에 사용한 출처만 포함하세요
- 사진 캡션(촬영 일시, 장소, [사진=출처])은 검증 대상이 아닙니다. 한국 뉴스는 과거 자료사진을 첨부하는 것이 일반적이므로, 캡션 정보가 기사 내용과 달라도 그것만으로 misleading 판단하지 마세요. 기사 본문의 핵심 사실 주장에만 집중하세요.
```

**After:**
```
## 중요: "확인 안 됨"과 "사실과 다름"은 완전히 다릅니다
- 검색에서 정보가 나오지 않는 것은 그 정보가 틀렸다는 뜻이 아닙니다
- "확인되지 않음" → unconfirmed (검색에 안 나옴, 출처를 못 찾음)
- "사실과 다름" → misleading (공식 출처가 기사와 반대되는 사실을 명시함)
- 이 구분을 반드시 지키세요

## 판단 기준
- verified: 공식 출처(정부, 기관, 공식 통계)로 핵심 내용이 뒷받침됨
- unconfirmed: 공식 출처를 찾을 수 없거나 충분히 확인되지 않음. 기사의 일부만 확인되고 나머지가 미확인이어도 unconfirmed입니다
- misleading: 공식 출처가 기사 내용과 명백히 모순될 때만 사용. "A라고 보도했지만 공식 발표는 B"처럼 반대되는 증거가 있어야 합니다

## 판단 우선순위
- 판단이 애매하면 반드시 unconfirmed을 선택하세요. misleading은 확실한 증거가 있을 때만 사용하는 최후의 수단입니다
- evidence에는 검색에서 실제 비교에 사용한 출처만 포함하세요
- 사진 캡션(촬영 일시, 장소, [사진=출처])은 검증 대상이 아닙니다. 한국 뉴스는 과거 자료사진을 첨부하는 것이 일반적이므로, 캡션 정보가 기사 내용과 달라도 그것만으로 misleading 판단하지 마세요
```
