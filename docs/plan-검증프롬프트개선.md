# Plan — 검증 프롬프트 개선: 사진 캡션 오판 방지

**Date:** 2026-03-02

---

## 문제

현재 `VERIFY_PROMPT_TEMPLATE`은 기사 본문에 포함된 **사진 캡션**(촬영 일시, 장소, 출처 표기)을 본문 사실의 일부로 취급한다.

한국 뉴스 관행상 **과거 자료사진을 현재 기사에 첨부하는 것은 일반적**이며, 캡션의 날짜/장소가 기사 내용과 다르다고 해서 기사가 왜곡인 것은 아니다.

**실제 사례:** 이란-미 항모 교전 기사에서 2025년 8월 샌디에이고 정박 사진이 첨부됨 → AI가 "항모 위치 정보 불일치"로 판단 → ❌왜곡/오류 태그 부여. 실제로는 기사 핵심 팩트(이란의 공격 주장, 미군의 반박)는 여러 언론이 동일하게 보도 중이므로 ⚠️사실 확인 중이 적절했음.

---

## 변경 대상

## 변경 내용

`## 판단 기준` 섹션(36행~)에 사진 캡션 관련 규칙 1줄 추가:

**Before:**
```
## 판단 기준
- verified: 공식 출처(정부, 기관, 공식 통계)로 핵심 내용이 뒷받침됨
- unconfirmed: 공식 출처를 찾을 수 없거나 충분히 확인되지 않음
- misleading: 공식 출처와 내용이 다르거나, 사실 오류 포함, 맥락 누락으로 오해 유도
- 확신이 없으면 반드시 unconfirmed으로 판단하세요
- evidence에는 검색에서 실제 비교에 사용한 출처만 포함하세요
```

**After:**
```
## 판단 기준
- verified: 공식 출처(정부, 기관, 공식 통계)로 핵심 내용이 뒷받침됨
- unconfirmed: 공식 출처를 찾을 수 없거나 충분히 확인되지 않음
- misleading: 공식 출처와 내용이 다르거나, 사실 오류 포함, 맥락 누락으로 오해 유도
- 확신이 없으면 반드시 unconfirmed으로 판단하세요
- evidence에는 검색에서 실제 비교에 사용한 출처만 포함하세요
- 사진 캡션(촬영 일시, 장소, [사진=출처])은 검증 대상이 아닙니다. 한국 뉴스는 과거 자료사진을 첨부하는 것이 일반적이므로, 캡션 정보가 기사 내용과 달라도 그것만으로 misleading 판단하지 마세요. 기사 본문의 핵심 사실 주장에만 집중하세요.
```
