# 리서치: 요약 난이도 문제 분석 — "쉬운 뉴스"와 "냉철한 팩트"의 균형

**Date:** 2026-03-02
**배경:** 프롬프트 개선(냉철한 톤, 직접 인용, 정확한 숫자) 이후 요약문이 "압축된 신문 기사"처럼 읽히는 문제 발생. 팩트렌즈의 핵심 가치 "AI가 뉴스를 쉬운 말로 요약"과 충돌.

## 1. 현재 출력 문제 분석 — test.md 9개 기사 해부

### 1.1 기사별 구체적 문제점

**기사 1: 국힘 필리버스터 중단 (정치)**

문제가 가장 심각한 기사. 구체적으로:

- **직접 인용 과다**: 송언석 원내대표의 발언이 3건 연속 직접 인용됨. "국회 법제사법위원회 거부 상황을 타개하기 위해 오늘 국민의힘에선 현시간 부로 필리버스터를 중단할 것을 결정했다" — 이 한 문장이 45자. 정치인의 공식 발언은 원래 복잡한 문장 구조를 갖고 있는데, 이를 원문 그대로 가져오니 독자가 소화해야 할 정보량이 폭증함.
- **숫자 과밀**: 지지율 28%, 전국 지지율 17%, 23~25일, ±3.1%포인트, 19시간, 26일 — 한 문단에 숫자 6개 이상이 밀집. 독자의 시선이 숫자 사이에서 길을 잃음.
- **전문용어 미풀이**: "필리버스터", "법제사법위원회", "국민투표법", "전국지표조사", "표본오차", "95% 신뢰수준" — 뉴스를 안 보는 20~30대 타겟이 이 용어들을 바로 이해할 수 있는가?
- **맥락 부족**: "왜 필리버스터를 했는지"가 빠져 있음. 행정통합법이 뭔지, 왜 중요한지 배경 설명 없음.
- **길이**: 3단락, 약 350자. "3~5문장"이라는 프롬프트 규칙을 문장 수로는 지키지만, 각 문장이 길어서 실질적 분량은 원래 기대의 2배.

**기사 2: 호르무즈 해협 (경제/국제)**

- 전문용어: "호르무즈 해협", "액화천연가스(LNG)", "수출화주", "살랄라·두쿰 항만", "물류비 바우처"
- 숫자 밀집: 70.7%, 20.4%, 50~80%, 3~5일, 1.9%, 136억 8천만 달러
- 직접 인용은 적지만, 기관명과 수치가 과다하여 읽기 어려움

**기사 3: 하메네이 사망 (국제)**

- 가장 긴 기사: 약 500자, 3단락
- 시간대·기관명 나열: "오전 9시 45분", "새벽 6시", "뉴욕타임스(NYT)", "CIA", "미군 중부사령부", "이란 국영 텔레비전", "이란 파르스 통신"
- "장엄한 분노 작전" 같은 고유명사가 설명 없이 등장
- 정보의 시간순 나열이 서사적 이해를 방해함 — "무엇이 일어났는지"는 알겠지만 "그래서 뭐가 중요한 건데?"가 빠져 있음

**기사 4: 이재명 대통령 대응 (정치)**

- 상대적으로 짧고 읽기 쉬움 (약 150자)
- 그러나 "김민석 국무총리", "강훈식 비서실장" 등 인물이 나열만 됨 — 왜 이 사람들이 중요한지 맥락 없음

**기사 5: 호르무즈 해협 봉쇄 (경제/국제)**

- 기사 2와 중복되는 주제
- "선박운행정보업체 케이플러", "일본총연구소", "바클레이즈 에너지 분석팀" — 일반 독자에게 이 기관들은 아무 의미가 없음
- "배럴당 100달러", "GDP가 3% 내려갈 가능성" — 숫자만 있고 "이게 독자에게 어떤 영향인지" 설명 없음

**기사 6: 국민의힘 사법개혁 비판 (정치)**

- 직접 인용이 기사의 70% 이상을 차지: "이재명 무죄 자판기", "사법 불신을 자초했다", "사법 3법으로 판사들을 고발 대상으로 만들고..."
- 정치인의 수사적 표현을 원문 그대로 인용하니, 요약이 아닌 "발언 중계"가 됨
- "법 왜곡죄", "재판소원제", "대법관 증원" — 3개의 법안명이 설명 없이 등장

**기사 7: 삼성 MWC (IT/경제)**

- 제품명·기능명 나열: "나우 넛지", "서클 투 서치", "프라이버시 디스플레이", "포토 어시스트", "갤럭시 버즈4", "갤럭시 북6", "갤럭시 XR", "갤럭시 Z 트라이폴드", "네트워크 인 어 서버"
- 528평, 2월 1일~5일 — 불필요한 세부 정보
- "폼팩터"라는 전문용어 미풀이

**기사 8: 정청래·장동혁 주택 (정치)**

- 직접 인용 밀집: "팔 게 많은 장 대표가 부럽다", "사실상 0주택자", "2억원도 채 안 되는 오피스텔은 팔려고 내놓아도 보러 오시는 분도 안 계신다", "이 대통령이 실제로 팔 줄은 몰랐다고 고백하시는 것이 어떠냐", "1억5000만원 정도면 팔릴 것"
- 5개의 직접 인용이 한 기사에 — 정치 공방의 "말"을 그대로 전달하니 독자가 "그래서 핵심이 뭔데?"라고 느끼게 됨
- 인물 4명(이재명, 장동혁, 정청래, 김현정, 최민희) — 누가 누구인지 정리 안 됨

**기사 9: 비트코인 (경제)**

- 상대적으로 읽기 쉬운 편
- 그래도 "6만 3천 달러", "3%", "6만 달러" — 달러 표기가 한국 독자에게 직관적이지 않음
- "코인데스크 아시아 데이터 및 토큰 팀 공동 리더 샤우랴 말와" — 이 사람이 누구인지는 독자에게 전혀 중요하지 않음

### 1.3 핵심 진단

**문제의 본질은 "차가움 vs 따뜻함"이 아니다.** 문제는 **"압축된 신문 기사 vs 풀어쓴 설명"**이다.

현재 프롬프트는 "냉철하게 쓰라" + "직접 인용하라" + "숫자 정확히 쓰라"를 결합했는데, Gemini가 이 세 규칙을 따르면서 **"풀어쓰기" 규칙을 무시**하게 됐다. 결과적으로:

- 정치인 발언 → 원문 그대로 인용 (풀어쓰기 불가)
- 숫자/통계 → 전부 포함 (선별 없음)
- 전문용어 → 인용문 안에 포함되면 풀어쓸 수 없음

직접 인용 규칙이 풀어쓰기 규칙을 **구조적으로 충돌**시킨 것이다.

## 2. 핵심 가치와의 충돌

### 2.1 팩트렌즈의 핵심 가치 정의

PRD 1.1절:
> "AI가 뉴스를 **쉬운 말로** 요약하고, 팩트체크 검증 태그를 붙여주며..."

PRD 1.3절의 1번 문제:
> "뉴스가 어렵다 — 전문용어, 배경설명 없이 던져주는 기사 → 읽다가 포기"

PRD 4.2절 요약 톤 가이드:
> "전문용어 사용 금지 — 어려운 개념은 일상 비유로 풀어쓰기"

린 캔버스의 가치가설:
> "20~30대 뉴스 이탈층은 AI가 **쉽게 요약**하고 팩트체크까지 해주는 뉴스 브리핑에 가치를 느낀다."

**"쉬운 말로 요약"은 팩트렌즈의 존재 이유 그 자체**다. 이것이 무너지면 서비스의 차별점이 사라진다.

### 2.2 "냉철하고 정확한 팩트"의 가치

프롬프트 개선의 배경은 정당했다:
- 이전 프롬프트의 "밝혔어요", "촉구했어요" 같은 가치 판단 동사는 중립성을 해침
- 존칭("두 분")은 특정 인물을 긍정적으로 프레이밍
- 간접 인용만 사용하면 원래 발언이 왜곡될 위험
- 숫자를 반올림하면 정확성이 떨어짐

이 문제들은 모두 해결해야 하는 실제 문제였다.

### 2.3 둘 다 달성할 수 있는가?

**가능하다.** 다만 현재 프롬프트처럼 "전부 직접 인용하라" + "전부 숫자 넣어라"는 방식으로는 불가능하다.

뉴닉이 증명하는 것: 쉬운 말로 정확한 뉴스를 전달할 수 있다. 핵심은 **정보의 선별과 변환**에 있다.

- 직접 인용을 "무조건 하라"가 아니라 "핵심 발언만 짧게 하라"
- 숫자를 "전부 넣어라"가 아니라 "가장 중요한 1~2개만 넣어라"
- 전문용어를 "그대로 쓰라"가 아니라 "괄호 안에 짧게 설명하라"

**"냉철함"과 "쉬움"은 별개의 축이다.** 냉철하면서도 쉬울 수 있고, 따뜻하면서도 어려울 수 있다. 현재 프롬프트는 "냉철함"을 추구하면서 실수로 "어려움"도 함께 가져온 것이다.

## 3. 이전 프롬프트 vs 현재 프롬프트 비교

### 3.1 프롬프트 텍스트 비교

| 요소 | 이전 프롬프트 | 현재 프롬프트 |
|------|-------------|-------------|
| **페르소나** | "뉴스를 쉽게 풀어서 설명해주는 뉴스 브리핑 작가" | "냉철한 뉴스 브리핑 작가. 사실만 건조하게 전달" |
| **문체** | 해요체, 3~5문장 | 해요체, 3~5문장 (동일) |
| **톤 규칙** | 없음 | "냉철하고 건조한 관찰자 시점" + 금지 동사 목록 |
| **존칭** | 규칙 없음 | "두 분", "~님" 금지 |
| **인용** | 규칙 없음 | "인물의 발언은 직접 인용하세요" |
| **숫자** | 규칙 없음 | "기사에 나온 그대로 정확히" + "약" 금지 |
| **전문용어** | "전문용어 사용 금지 — 일상적인 비유로 풀어쓰기" | "전문용어는 일상적인 비유로 풀어쓰세요" |
| **배경 설명** | "이게 왜 중요하냐면" 식으로 깔기 | "이게 왜 중요하냐면" 식으로 깔기 (동일) |

### 3.2 출력 품질 비교

| 평가 항목 | 이전 출력 | 현재 출력 |
|-----------|-----------|-----------|
| **쉬움** | 높음 — 비유, 풀어쓰기 활발 | 낮음 — 압축된 신문체 |
| **정확성** | 낮음 — 숫자 반올림/누락, 간접 인용만 | 높음 — 숫자 정확, 직접 인용 포함 |
| **중립성** | 낮음 — "밝혔다/촉구했다" 등 가치 판단 동사, 존칭 | 높음 — "말했다" 통일, 존칭 없음 |
| **배경 맥락** | 있음 — "왜 중요하냐면" 설명 포함 | 거의 없음 — 사실 나열에 집중 |
| **읽는 시간** | 짧음 — 짧은 문장, 풀어쓰기 | 김 — 긴 인용문, 숫자 밀집 |
| **타겟 적합성** | 높음 — 20~30대 뉴스 초보에게 적합 | 낮음 — 이미 뉴스를 읽는 사람에게나 적합 |
| **감정적 톤** | 따뜻함 — "~거예요", 해석적 표현 | 건조함 — "말했어요" 반복 |

### 3.3 각각의 구체적 문제

**이전 프롬프트의 문제 (해결 필요했던 것들):**
1. "밝혔어요" → 발언의 중요도를 편집자가 암시
2. "촉구했어요" → 화행(speech act)에 대한 해석
3. "두 분" → 특정 인물에 대한 존경 암시
4. 숫자 반올림 → "약 30%" 같은 표현이 팩트체크 서비스에서 신뢰도 저하
5. 간접 인용만 사용 → 발언 왜곡 가능성

**현재 프롬프트의 문제 (새로 생긴 것들):**
1. 직접 인용 과다 → 정치인 발언의 복잡한 문장 구조가 그대로 들어옴
2. 숫자 전량 포함 → 정보 과부하, 스캔 불가
3. 배경 설명 후퇴 → "건조하게 전달"이라는 페르소나가 "풀어쓰기"를 억제
4. 인물/기관 나열 → 누가 누구인지 맥락 없이 이름만 등장
5. 전문용어 풀이 미실행 → "일상 비유로 풀어쓰세요" 규칙이 있지만, "냉철하게/건조하게"라는 상위 지시가 이를 억압

### 3.4 프롬프트 간 충돌 지점

현재 프롬프트에는 **내부 모순**이 있다:

- "전문용어는 일상적인 비유로 풀어쓰세요" vs "인물의 발언은 직접 인용하세요"
  → 인용문 안의 전문용어는 풀어쓸 수 없음
- "3~5문장으로 핵심만 요약" vs "숫자는 기사에 나온 그대로 정확히" + "출처가 명확한 통계나 수치가 있으면 반드시 포함"
  → 숫자를 전부 넣으면 3~5문장을 넘기거나, 문장이 극도로 길어짐
- "배경 맥락이 필요하면 '이게 왜 중요하냐면' 식으로 깔아주세요" vs "냉철하고 건조한 관찰자 시점"
  → "이게 왜 중요하냐면"은 설명자 모드인데, "건조한 관찰자"는 설명을 최소화하려 함

LLM은 모순되는 지시를 받으면 **나중에 나온 규칙**이나 **페르소나와 더 일치하는 규칙**을 우선한다. "냉철한 관찰자"라는 페르소나가 설정되면, "풀어쓰기"와 "배경 설명" 규칙이 자연스럽게 약해진다.

## 4. 벤치마크 분석 — MZ 뉴스레터의 "쉬우면서도 정확한" 균형

### 4.1 뉴닉의 접근법

뉴닉(구독자 110만)의 핵심 전략은 **"번역"**이다. 뉴스를 "쉬운 말로 번역"하는 것이지, "뉴스를 그대로 전달"하는 것이 아니다.

**직접 인용 처리:**
- 뉴닉은 직접 인용을 **극히 제한적으로** 사용한다
- 정치인 발언을 그대로 인용하기보다는, 발언의 핵심 의미를 풀어서 전달
- 직접 인용을 쓸 때도 **핵심 단어/구만 짧게** 인용: "검찰이 '~~하자'라고 했어"
- 긴 공식 발언을 통째로 인용하지 않음
- 나무위키 설명: "주체가 사람인 것처럼 말을 하듯이 표현한다"

**숫자 처리:**
- 핵심 숫자 1~2개만 선별
- "약 70%"처럼 가독성을 위한 반올림 허용
- 맥락 없는 숫자 나열 회피
- 숫자 뒤에 "이게 뭘 의미하냐면" 식의 해석 추가

**전문용어 처리:**
- 망 사용료 → "고속도로 통행요금 같은 거예요"
- "조금이라도 어려운 용어는 추가설명을 덧붙인다" (나무위키)
- 전문용어를 쓸 수밖에 없을 때는 괄호 안에 일상어 번역 제공

**구조:**
- 짧은 문장 + 짧은 문단
- 독자의 질문을 대신 던져주는 방식 ("그래서 이게 왜 중요하냐면~")
- 5~10분 모바일 최적화 분량

### 4.2 어피티의 접근법

어피티의 핵심은 **"친절한 경제 멘토"** 포지셔닝이다.

- 대표 박진영: "무시받는 것 같은 인상이 조금이라도 들지 않게 말투를 신경 쓴다"
- 비공식 사훈 "투신자판"(투자는 신중하게 자신의 판단으로) — 객관적이면서 친절
- 이모지를 체계적으로 사용하여 시각적 스캔을 도움
- 경제 용어를 일상 비유로 번역: "아무리 몸에 좋은 음식도 맛이 없으면 꾸준히 먹기 어렵듯"

### 4.3 토스 AI 시그널의 접근법

토스는 AI 뉴스에서 유일한 톤 혁신 사례:
- UX 라이팅 5대 원칙: Clear(명확), Concise(간결), Casual(친근), Respect(존중), Emotional(공감)
- "5000원 송금이 완료되었습니다" → "5000원을 보냈어요" — 같은 정보, 다른 체감
- 뉴스+공시 데이터를 결합해 "왜 올랐을까?" → "공급망 회복과 실적 개선 기대 때문"이라는 해석적 변환

### 4.4 벤치마크에서 도출한 공통 원칙

MZ 뉴스 서비스들이 공유하는 "쉬우면서도 정확한" 균형 전략:

| 원칙 | 구체적 방법 | 팩트렌즈 현재 상태 |
|------|-----------|------------------|
| **인용은 짧게, 핵심만** | 긴 발언 → 핵심 구절 3~10단어만 직접 인용, 나머지는 간접 인용으로 풀어쓰기 | 전체 발언을 통째로 직접 인용 |
| **숫자는 선별** | 가장 영향력 있는 숫자 1~2개만 남기고, 나머지는 "크게 올랐어요" 같은 정성적 표현 | 기사의 모든 숫자를 포함 |
| **이름보다 역할** | "코인데스크 분석가" → "암호화폐 분석가", 불필요한 고유명사 제거 | 기관명·인명을 모두 나열 |
| **전문용어 = 설명 의무** | 용어 사용 시 반드시 괄호 또는 다음 문장에서 풀이 | 규칙은 있으나 실행 안 됨 |
| **맥락 먼저** | "이게 왜 중요하냐면" → 사실 나열 순서 | 사실 나열 → 맥락 없음 |
| **짧은 호흡** | 한 문장 15~25자 내외, 한 문단 2~3문장 | 한 문장 40~60자, 인용문 포함 시 더 김 |

### 4.5 Plain Language 국제 표준의 시사점

미국 Digital.gov의 Plain Language Guide와 NN/g의 연구에 따르면:

- **숫자 표현**: "Three in four participants"가 "three-quarters of participants"보다 직관적 — **비율보다 비유가 쉬움**
- **전문용어**: "name and explain" 기법 — 용어를 쓰되 바로 다음에 쉬운 말로 설명
- **인용**: 긴 인용은 피하고, 핵심 메시지를 paraphrase(바꿔쓰기) — **요약에서 원문 그대로의 인용은 가독성을 해침**
- **문단**: 3~5문장, 한 문장은 20단어(한국어로 약 15~25자) 이내가 이상적

## 5. 구체적 개선 방향 제안

### 5.1 근본 진단: 프롬프트의 규칙 우선순위 문제

현재 프롬프트에는 규칙이 있지만 **우선순위가 없다**. LLM은 "직접 인용하라"와 "쉽게 쓰라"가 충돌할 때 어느 것을 우선할지 판단할 수 없다. 해결책은 **규칙에 명시적 우선순위를 부여**하는 것이다.

제안하는 우선순위:
1. **쉬움** (최우선) — 20대도 한 번에 이해할 수 있어야 함
2. **정확성** — 사실을 왜곡하지 않아야 함
3. **중립성** — 편향 없이 전달해야 함
4. **완결성** — 맥락이 포함되어야 함

이 우선순위에서 "쉬움"이 1순위이므로, 직접 인용이 쉬움을 해치면 간접 인용으로 바꿔야 한다. 숫자가 너무 많으면 핵심만 남겨야 한다.

### 5.2 직접 인용 규칙 개선

**현재**: "인물의 발언은 직접 인용하세요"
**문제**: 모든 발언을 통째로 인용 → 정치인의 복잡한 공식 발언이 그대로 들어옴

**제안**: 직접 인용은 **핵심 구절(5~15자)만**, 나머지는 간접 인용으로 풀어쓰기

예시:

현재 출력:
> 송언석 국민의힘 원내대표는 1일 기자회견에서 "국회 법제사법위원회 거부 상황을 타개하기 위해 오늘 국민의힘에선 현시간 부로 필리버스터를 중단할 것을 결정했다"고 말했어요.

개선 방향:
> 국민의힘 송언석 원내대표는 법사위가 막혀 있는 상황을 풀기 위해 "필리버스터를 중단하겠다"고 말했어요.

변화: 45자 인용 → 10자 인용 + 간접 인용으로 맥락 풀어쓰기

### 5.3 숫자 규칙 개선

**현재**: "숫자는 기사에 나온 그대로 정확히" + "출처가 명확한 통계나 수치가 있으면 반드시 포함"
**문제**: 기사의 모든 숫자가 포함됨 → 정보 과부하

**제안**: 핵심 숫자는 1~2개만 선별. 나머지는 정성적 표현으로 대체.

예시:

현재 출력:
> 대구·경북의 국민의힘 지지율은 28%로 더불어민주당과 동률을 기록했으며, 국민의힘 전국 지지율은 17%로 떨어졌어요. 이 조사는 지난 23일부터 25일까지 실시되었고, 표본오차는 95% 신뢰수준에 ±3.1%포인트예요.

개선 방향:
> 대구·경북에서 국민의힘 지지율이 28%까지 떨어져 민주당과 동률이 됐어요. 전통적 지지 기반에서도 여론이 악화된 거예요.

변화: 숫자 6개 → 1개 + 맥락 해석 추가. 조사 기간/표본오차는 팩트체크 영역의 역할이지 요약의 역할이 아님.

### 5.4 전문용어 규칙 강화

**현재**: "전문용어는 일상적인 비유로 풀어쓰세요" (규칙은 있지만 실행 안 됨)
**문제**: "냉철한 관찰자" 페르소나가 풀어쓰기를 억제

**제안**: 전문용어 풀이를 **의무적 형식**으로 지정

형식: `전문용어(쉬운 설명)` 또는 전문용어 대신 쉬운 말로 대체

예시:

현재 출력:
> 필리버스터를 중단했어요

개선 방향:
> 필리버스터(법안 통과를 막기 위한 무제한 토론)를 중단했어요

또는:
> 법안 통과를 막기 위해 진행하던 무제한 토론을 중단했어요

### 5.5 페르소나 재설계

**현재**: "냉철한 뉴스 브리핑 작가. 사실만 건조하게 전달하세요."
**문제**: "건조하게 전달"이 "풀어쓰기"와 "배경 설명"을 억제

**제안**: 페르소나를 "냉철하지만 쉽게 설명하는" 방향으로 재설계

예시:
> "당신은 냉철하면서도 쉽게 설명하는 뉴스 브리핑 작가입니다. 20대가 한 번에 이해할 수 있도록 쉬운 말로 쓰되, 감정이나 의견은 넣지 마세요."

핵심 변화: "건조하게 전달" → "쉬운 말로 쓰되 감정/의견은 넣지 마세요"

이렇게 하면 "쉬움"과 "중립성"을 동시에 달성하면서, "건조하게"라는 모호한 지시가 풀어쓰기를 억제하는 문제를 해결할 수 있다.

### 5.6 인물/기관 나열 규칙 추가

**현재**: 규칙 없음
**문제**: "코인데스크 아시아 데이터 및 토큰 팀 공동 리더 샤우랴 말와" 같은 불필요한 고유명사

**제안**:
- 핵심 당사자(대통령, 당 대표 등) 외의 인물은 역할/직책으로만 표기
- 기관명은 독자가 아는 수준으로 단순화: "한국무역협회" → "무역협회", "선박운행정보업체 케이플러" → "해운 분석 업체"
- 한 기사에 인물은 최대 2~3명만

### 5.7 구조적 개선: "맥락 먼저" 원칙 강화

**현재**: "배경 맥락이 필요하면 '이게 왜 중요하냐면' 식으로 깔아주세요"
**문제**: "필요하면"이라는 조건부 표현 → LLM이 거의 실행하지 않음

**제안**: 조건부가 아닌 필수 구조로 변경

요약문 구조:
```
1문장: 무슨 일이 일어났는지 (핵심 사실)
2문장: 왜 이런 일이 일어났는지 (배경/원인)
3~4문장: 구체적 내용 (선별된 인용, 핵심 숫자)
5문장 (선택): 앞으로 어떻게 될지 (전망/영향)
```

### 5.8 종합 — 프롬프트 규칙 제안 (초안)

기존 "인용과 숫자" 섹션의 규칙을 다음과 같이 재설계:

```
## 인용
- 인물의 발언에서 핵심 구절(5~15자)만 직접 인용("").
  긴 발언은 쉬운 말로 바꿔서 간접 인용하세요.
- 나쁜 예: "국회 법제사법위원회 거부 상황을 타개하기 위해
  현시간 부로 필리버스터를 중단할 것을 결정했다"고 말했어요.
- 좋은 예: 법사위가 막힌 상황을 풀기 위해 "필리버스터를
  중단하겠다"고 말했어요.

## 숫자
- 핵심 숫자 1~2개만 포함하세요. 나머지는 "크게 올랐어요",
  "절반 이상" 같은 쉬운 표현으로 바꾸세요.
- 조사 기간, 표본오차, 부차적 통계는 생략하세요.
- 포함한 숫자는 정확해야 합니다 — 반올림하지 마세요.

## 인물/기관
- 핵심 당사자 2~3명 외의 인물은 역할로만 표기하세요.
  (예: "암호화폐 분석가", "야당 대변인")
- 기관명은 짧게 줄이세요.
  (예: "한국무역협회" → "무역협회")
```

## 6. 결론

### 6.1 문제의 본질

팩트렌즈의 현재 문제는 "차가운 톤"이 아니다. **"정보 압축 vs 정보 번역"**의 문제다.

- 이전 프롬프트: 정보를 **번역**함 (쉽지만 부정확)
- 현재 프롬프트: 정보를 **압축**함 (정확하지만 어려움)
- 필요한 것: 정보를 **선별하고 번역**함 (쉬우면서도 정확)

### 6.2 핵심 인사이트

1. **직접 인용 = 쉬움의 적**. 정치인의 공식 발언은 태생적으로 복잡한 문장이다. 이를 통째로 가져오면 아무리 해요체로 써도 어렵다. 해결: 핵심 구절만 짧게 인용.

2. **숫자 과다 = 스캔 불가**. 한 문단에 숫자 3개 이상이면 독자의 시선이 멈춘다. 해결: 가장 중요한 1~2개만 남기고 나머지는 정성적 표현.

3. **페르소나가 규칙을 이김**. "냉철한 관찰자"라는 페르소나가 "풀어쓰기" 규칙을 구조적으로 억압한다. 해결: 페르소나를 "냉철하면서도 쉽게 설명하는"으로 재설계.

4. **규칙 간 우선순위 부재**. "직접 인용하라"와 "쉽게 쓰라"가 충돌할 때 LLM은 판단할 수 없다. 해결: "쉬움 > 정확성 > 중립성" 우선순위 명시.

5. **"냉철함"과 "쉬움"은 양립 가능**. 뉴닉은 쉬우면서도 팩트 중심이고, 토스는 친근하면서도 정확하다. "감정을 빼라"와 "어렵게 써라"는 다른 말이다.
