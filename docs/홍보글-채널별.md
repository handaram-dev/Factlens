> 디스콰이엇은 "문제 → 솔루션 → 차별점" 구조. 메이커/개발자 대상이므로 기술적 맥락 포함.

**소개글:**
```
## 문제

20~30대의 뉴스 이탈이 심각합니다.

뉴스는 전문용어로 가득하고, 배경 설명 없이 던져지고,
어떤 언론사를 믿어야 할지 알 수 없습니다.

## 솔루션

팩트렌즈는 매일 아침 7시에 자동으로 뉴스 브리핑을 생성합니다.

1. 쉬운 요약
   - Google News RSS에서 국내·경제·국제 3개 토픽의 뉴스를 수집합니다
   - 최대 15개 기사를 선별한 뒤, 각 기사의 본문을 스크래핑합니다
   - Gemini Flash가 20대도 이해할 수 있는 쉬운 말로 요약합니다
   - 전문용어는 괄호 안에 설명을 붙여줍니다
     예: "필리버스터(법안 통과를 막는 무제한 토론)"

2. 팩트체크 검증 태그
   - Gemini에 Google Search Grounding을 연결해서
     각 기사의 내용을 공식 출처(정부 발표, 기관 보도자료, 공식 통계)와 교차확인합니다
   - 검증 결과에 따라 3가지 태그를 자동 부여합니다:
     ✅ 사실 확인됨 — 공식 출처로 뒷받침됨
     ⚠️ 사실 확인 중 — 충분히 확인되지 않음
     ❌ 왜곡·오류 — 공식 출처와 내용이 다름
   - 판별 근거와 출처 링크도 함께 제공합니다

3. 다양한 시각
   - 같은 사건을 보도한 여러 언론사의 원문 링크를 나란히 보여줍니다

## 기술 스택

- Python 3.12
- Gemini Flash (요약) + Gemini + Google Search Grounding (검증)
- newspaper3k + trafilatura (본문 스크래핑, 2중 폴백)
- GitHub Actions (매일 KST 07:00 자동 실행)
- Cloudflare Pages (정적 호스팅)
- 운영 비용: 거의 ₩0/월

## 7. 긱뉴스 — Show GN

> 긱뉴스는 기술적 깊이를 중시. 제목에 핵심 가치 + 기술적 키워드 포함.

**제목:** Show GN: 팩트렌즈 — Gemini Search Grounding으로 뉴스 팩트체크를 자동화한 사이드프로젝트

**본문:**
```
팩트렌즈(factlens.pages.dev)는 매일 아침 자동으로 뉴스를 수집하고,
쉬운 말로 요약하고, 팩트체크 검증 태그를 붙이는 뉴스 브리핑 서비스입니다.

## 동기

뉴스 팩트체크를 자동화할 수 있을까? 라는 질문에서 시작했습니다.
SNU 팩트체크센터가 2024년 폐쇄된 이후, 한국에는 대중이 쉽게 접근할 수 있는
팩트체크 서비스가 사실상 없습니다.

## 파이프라인

매일 KST 07:00에 GitHub Actions가 6단계 파이프라인을 실행합니다:

1. 수집: Google News RSS 3개 토픽(국내/경제/국제) 파싱 (feedparser)
2. 선별: 피드 간 라운드 로빈 + 중복 제거 → 최대 15개 기사
3. 요약: 기사 본문 스크래핑(newspaper3k → trafilatura 폴백) + Gemini Flash로 요약 생성
   - 파싱 실패 감지 3층 방어: 면책 키워드 필터 → 최소 길이 체크(200자) → 프롬프트 내 [[INVALID]] 규칙
4. 검증: Gemini + Google Search Grounding으로 교차확인
   - 공식 출처(정부 발표, 기관 보도자료 등)와 대조
   - verified / unconfirmed / misleading 3종 태그 부여
   - grounding_metadata에서 evidence 링크 + search_entry_point 자동 추출
5. 렌더링: Jinja2로 정적 HTML 생성 + sitemap.xml/robots.txt 자동 생성
6. 배포: dist/ 디렉토리를 git push → Cloudflare Pages 자동 배포

## 기술적으로 재밌었던 부분

- Gemini Google Search Grounding이 핵심입니다.
  처음엔 Google Custom Search API로 직접 검색해서 비교했는데,
  Grounding을 쓰면 Gemini가 직접 Google 검색을 실행하고
  검색 결과를 근거로 판단까지 해줍니다.
  grounding_metadata에서 출처 URL과 검색 위젯 HTML까지 뽑아올 수 있어서
  별도 검색 API 비용이 0입니다.

- 요약 프롬프트를 상당히 많이 다듬었습니다.
  "20대도 이해할 수 있게" 쓰는 게 생각보다 어렵습니다.
  우선순위를 "쉬움 > 정확성 > 중립성"으로 명시하고,
  전문용어 괄호 설명 의무화, 인용 15자 제한, 숫자 1~2개만 허용 등
  구체적인 규칙을 많이 넣었습니다.

- newspaper3k의 파싱 실패가 꽤 많았습니다.
  이용약관이나 면책조항을 기사 본문으로 파싱하는 경우가 있어서
  키워드 필터 + 최소 길이 체크 + 프롬프트 내 감지 규칙을 3중으로 넣었고,
  trafilatura를 폴백 파서로 추가했습니다.
